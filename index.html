安装步骤

bash 
# 一键安装
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash

# 启动配置向导 — 设置模型和 API key
hermes setup

# 你的第一句对话
hermes chat -q "你好，帮我写一个 Hello World Python 脚本"

# 进入交互模式（在这里持续对话）
hermes

需要理解的概念

  • hermes chat -q "..." = 一次性问答，聊完退出
  • hermes（无参数）= 交互模式，持续对话
  • hermes setup = 配置向导，帮你选模型、填 API key

深入理解配置和模型
约 1 周 · 难度：中级
▾
目标：理解 Hermes 的配置体系，能自由切换模型，能根据自己的需求微调行为。

常用管理命令

bash 
hermes model          # 交互式切换模型
hermes config         # 查看当前完整配置
hermes config edit    # 用编辑器打开 config.yaml
hermes doctor         # 健康检查（依赖和配置）
hermes logs --follow  # 实时查看日志
hermes status         # 组件状态一览

必知的配置项

yaml (~/.hermes/config.yaml) 
model:
  default: "deepseek/deepseek-chat"   # 默认模型
  provider: "openrouter"              # 提供商

agent:
  max_turns: 90                       # 最大工具调用轮数

terminal:
  backend: "local"                    # 本地执行
  timeout: 180                        # 命令超时（秒）

display:
  skin: "default"                     # 界面主题
  show_cost: true                     # 显示 token 费用

关键概念

  • Provider（提供商）与 Model（模型） — Provider 是 API 入口（如 OpenRouter、DeepSeek），Model 是具体的 LLM
  • config.yaml 管行为，.env 管密钥 —— 永远不要把 API key 写进 config.yaml
  • 每个模型有不同的价格和能力（代码、推理、速度），要根据任务选择
  • 支持的 Provider 超过 20 个，覆盖国内外主流 LLM

记忆系统

Hermes 能在你允许的情况下记住重要信息，跨会话保存。下次对话时自动加载。试试说：

子任务委托（delegate_task）

把复杂任务拆分给「子代理」并行执行。每个子代理有独立的上下文和终端。

对话示例 
"帮我同时研究 Flask 和 FastAPI 的最新版本，比较它们的差异"

"把 tests/ 目录分成 4 组，并行运行每组测试，汇总结果"
  • 单个子代理：适合需要深度推理的子任务
  • 批量并行：最多 3 个子代理同时运行（可配置）
  • 子代理之间完全隔离，不影响主对话

后台任务

对话示例 
/background 帮我运行完整的测试套件

/background 帮我构建 Docker 镜像并推送

/stop    停止所有后台进程
/agents  查看活跃的代理和任务

定时任务（Cron）

bash 
# 每天早上 9 点汇总科技新闻
hermes cron create "0 9 * * *" \
  --prompt "搜索今天科技新闻，总结 5 条最重要的"

# 每 2 小时检查一次服务器状态
hermes cron create "every 2h" \
  --prompt "检查 myapp.com 是否正常响应"

# 管理定时任务
hermes cron list            # 列出所有
hermes cron pause <ID>     # 暂停
hermes cron run <ID>       # 立即触发
hermes cron remove <ID>    # 删除

写自定义工具（插件）

在 ~/.hermes/plugins/ 下创建插件，添加你自己的工具：

text 
插件目录结构：
~/.hermes/plugins/my-plugin/
├── plugin.yaml        # 插件元数据
└── __init__.py        # 工具实现
python (__init__.py 示例) 
import json
from tools.registry import registry

def my_tool(param: str, task_id: str = None) -> str:
    return json.dumps({"success": True, "data": param})

registry.register(
    name="my_tool",
    toolset="my_toolset",
    schema={
        "name": "my_tool",
        "description": "我的自定义工具",
        "parameters": {
            "type": "object",
            "properties": {
                "param": {"type": "string", "description": "参数"}
            },
            "required": ["param"]
        }
    },
    handler=lambda args, **kw: my_tool(
        param=args.get("param", ""), task_id=kw.get("task_id")),
)

❓ 常见问题

工具不生效 / 找不到某个工具 ▾

三个原因最常见：

1. 工具集没启用 — 运行 hermes tools 检查，确保对应平台勾选了该工具

2. 改了配置没重启 — 工具变更需要 /reset 开始新会话才生效

3. 缺环境变量 — 部分工具需要 API key，检查 ~/.hermes/.env

改了配置为什么不生效？ ▾

不同类型改动的生效方式不同：

· 工具开关 — 需要 /reset 新会话

· 模型/Provider — 需要 /reset 或退出重启

· Gateway 配置 — 需要 hermes gateway restart

· 环境变量 (.env) — 完全退出 Hermes 后重新启动

模型返回 403 / 401 错误 ▾

401：API key 无效或过期。运行 hermes doctor 检查，重新设置 .env 里的密钥。

403：权限不足。常见于 OAuth 登录过期，运行 hermes login 重新授权。

Copilot 用户注意：gh auth login 的 token 不能用于 Copilot API，必须用 hermes model → GitHub Copilot 的专用 OAuth 流程。

怎么让 Hermes 记住我的习惯？ ▾

直接在对话里说：

"记住我用 pytest 不用 unittest"

"记住我的项目默认用 Python 3.12"

"记住我在 UTC+8 时区"

这些会存入持久记忆，下次对话自动加载。查看记忆：hermes memory status。

怎么省 Token / 省钱？ ▾

· 选对模型 — 写代码用 DeepSeek/Claude，翻译用 Gemini Flash，不要大材小用

· 减少工具集 — hermes tools 关掉不用的工具，每次 API 调用会少很多 token

· 适时 /new — 对话太长上下文会爆掉，定期 /new 开始新会话

· 开启压缩 — hermes config set compression.enabled true，长对话自动精简

· 看费用 — /usage 查看当前会话和总计花费

Gateway 连不上 / 总是挂掉 ▾

先看日志：grep -i "error\|failed" ~/.hermes/logs/gateway.log | tail -20

SSH 退出就挂：sudo loginctl enable-linger $USER

WSL2 窗口关了就挂：确保 /etc/wsl.conf 里有 [boot] systemd=true

崩溃循环：systemctl --user reset-failed hermes-gateway

Discord 静默：Bot 设置里必须开启 Message Content Intent

语音 / 生图 / Spotify 不工作 ▾

这些都不是开箱即用的功能，需要额外配置：

· 语音输入 (STT) — 需要 pip install faster-whisper 或设 API key

· 文字转语音 (TTS) — 默认用免费 Edge TTS，无需配置

· 生图 (ComfyUI/SD) — 需要 GPU（≥6GB VRAM）或 Comfy Cloud 付费

· Spotify — 需要 hermes auth spotify 授权

· 智能灯 (Hue) — 需要 Hue 桥接器 + openhue CLI + 局域网配对
